1️⃣ Tagging & Categorization
Goal: Allow users to organize documents into categories or add custom tags.

Implementation Steps:

DB Change: Add a tags TEXT column (comma-separated or JSON) or create a document_tags join table for many-to-many.

Frontend:

Show a “Tags” column in the table.

Allow editing tags inline or via a small modal.

Filter/search by tags.

Optional: Add color-coded tag chips.

2️⃣ Export / Import Database
Goal: Backup and transfer document library.

Implementation Steps:

Export:

Copy database.sqlite to a user-selected location.

Or export JSON for cloud-friendly sync.

Import:

Replace current DB with selected file.

Trigger refresh (documents-updated).

Bonus: Merge two DBs to avoid overwriting.

3️⃣ Drag & Drop UX Improvements
Highlight drop area when dragging files/folders.

Show a toast or badge for “X files added”.

4️⃣ Optional Enhancements
File Preview: Show PDF thumbnails or image previews in a side panel.

Watch Folders: Automatically detect new files in selected folders (using chokidar).

Sorting & Columns: Allow sorting by date, size, name.

Multi-select actions: Delete, open multiple files at once.
